### 2. `/chat` route

Layout:
- Full-height flex column.
- Top: minimal header with "Rezept-Chat" + back button.
- Middle: message list (scrollable).
- Bottom: text input + send button (sticky).

Message bubbles:
- User messages right-aligned sage primary.
- Assistant messages left-aligned neutral.
- Typing indicator while awaiting response.
- Error message (inline, red) on network failure + "Erneut senden" button.

### 3. Session management

- `sessionId` generated client-side on mount (`crypto.randomUUID()`), stored in URL `?session=…` so refresh preserves it.
- Full `messages[]` sent on every turn (backend is stateless per P2-4).
- Client-side cap: UI warns at 25 turns, blocks send at 30 (matches P2-4's 413).

### 4. "In Rezept umwandeln" button

- Appears after the assistant has sent at least 2 messages (indicating a recipe has been discussed).
- Click → `POST /api/chat/:sessionId/to-recipe` with the full `messages[]`.
- Receives `ExtractionResult` → navigate to `/groups/:groupId/recipes/new?chatImportId=<transient-client-id>` and stash the result in `sessionStorage`. `RecipeFormPage` reads from `sessionStorage` when `chatImportId` is present.
- Group picker modal if user has > 1 group.

### 5. Tests

Web:
- `ChatPage.test.tsx`: user types + sends → mutation called → assistant bubble appears.
- Optimistic append: user bubble renders before network round-trip completes.
- Turn cap: sending 31st message blocked with UI warning.
- Scroll-to-bottom: new message arrives → list scrolls; user-scrolled-up state sticky.
- "In Rezept umwandeln": after N turns, button appears, click navigates + stashes data.
- Error + retry: network failure → inline error → click retry resends.

## Non-goals

- Streaming responses (SSE) — v1.1 polish.
- Persistent chat history per user — sessions live only in URL + sessionStorage.
- Multi-user conversations.
- Voice input.
- Image attachments in chat (user uploads → that's photo-import path).

## Acceptance criteria

- Web gates green.
- `pnpm build` clean.
- Manual: have a 3-turn dialogue → convert to recipe → save → recipe visible.

## Anti-shortcut reminders

- TDD every UX state.
- Optimistic append has rollback on send-failure (don't leave a ghost user bubble).
- Turn cap at 30 is a hard limit; warn at 25.
- `sessionStorage` stash cleaned after successful save OR explicit "verwerfen".
- Don't persist messages to `localStorage` (privacy: chat may mention medical/dietary info).
